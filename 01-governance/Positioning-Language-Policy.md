# Positioning and Language Policy

## Purpose
Define approved language for public documentation to prevent overstatement of authority, endorsement, or compliance outcomes.

## Standard Disclaimer
Use this statement in public-facing entry-point documents:

This repository is an independent, experimental governance-as-code project. It is not affiliated with or endorsed by Gartner and does not provide certification, legal advice, or compliance guarantees.

## Approved Terms
- conceptual operating model reference
- alignment
- mapping
- governance-as-code pattern
- template
- example
- organization-specific implementation

## Restricted Terms and Claims
Do not use these phrases in a way that implies authoritative status:
- TRiSM certified
- TRiSM certification
- guaranteed compliance
- compliant by default
- official Gartner implementation

## Wording Rules
- Use alignment language instead of compliance claims.
- State artifacts as templates and examples that require adaptation.
- Keep regulatory statements informational unless legal review is explicitly documented.
- Avoid superlative or definitive claims for framework superiority.

## PR Review Expectation
All documentation pull requests should include reviewer confirmation that positioning and language rules are respected.
