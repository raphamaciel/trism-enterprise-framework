# Exception Register Template

## Purpose
Track approved control exceptions with clear ownership, rationale, and time-bound review conditions.

| Exception ID | Date Raised | Control ID | Exception Description | Business Justification | Risk Impact | Exception Owner | Approver | Compensating Control | Expiry Date | Review Criteria | Next Review Date | Status |
|---|---|---|---|---|---|---|---|---|---|---|---|---|

## Status Values
- Open
- Approved
- Expired
- Closed

## Governance Rules
- Every exception must have an owner.
- Every exception must include a business justification.
- Every exception must include an expiry date and review criteria.
- Expired exceptions must be escalated or renewed with new approval.
